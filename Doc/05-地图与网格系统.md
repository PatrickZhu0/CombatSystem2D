# 地图与网格系统实现

## 系统架构总览

地图系统由多层网格叠加构成，各层职责不同：

```
┌─────────────────────────────────────┐
│  视觉渲染层 — MapTile (瓦片渲染)     │
├─────────────────────────────────────┤
│  碰撞通行层 — PassageRect (通行检测)  │  ← 比瓦片更粗的分辨率
├─────────────────────────────────────┤
│  寻路层 — StaticGrid / Node (A*)     │  ← 精细的寻路网格
├─────────────────────────────────────┤
│  空间分区层 — nearGridPosList (碰撞)  │  ← 物体查询加速
├─────────────────────────────────────┤
│  房间结构层 — GridScript / MazeScript │  ← 副本迷宫拓扑
└─────────────────────────────────────┘
```

## 3. 通行矩形网格（Passage Rect Grid）

碰撞通行使用**独立于瓦片的矩形网格**，分辨率更粗（通常 4-8 个像素一格）：

### MapManager 中的关键字段（line 459414）

```csharp
public Byte[] MapTileAttribute_Array;    // 压缩的通行属性数组
public Int32 mapPassageRectWidth;        // 通行网格宽度
public Int32 mapPassageRectHeight;       // 通行网格高度
public Int32 mapTileWidth, mapTileHeight; // 地图瓦片尺寸
public Int32 baseTileYCount;             // 基础行数
public Single mapBasicTileFloorHeight;   // 地板碰撞高度
```

### 通行属性

每个单元格存储一个 `Byte`，编码 `ENUM_PASSAGETYPE` 标记：

```
索引计算: array[passageRectY * mapPassageRectWidth + passageRectX]

Byte 值 → 通行标记:
  · 地面单位是否可通行
  · 飞行单位是否可通行
  · 单向平台
  · 完全阻挡
```

### 查询与更新方法

```csharp
GetPassageRect(int x, int y)                      // 查询通行性
SetPassageRect(int x, int y, Byte passageType)     // 设置通行性
UpdatePassageRect()                                // 重新计算（物体阻挡变化时）
```

## 5. 空间分区（碰撞加速）

### IRDCollisionObject 中的网格查询（line 415456）

```csharp
private static List<GridPos> nearGridPosList;    // 邻近网格列表
private const Int32 MAX_NEAR_GRID_COUNT;         // 最大查询范围
private const Int32 MAX_NEAR_GRID_COUNT_HALF;
```

### 查询方法

```csharp
InitNearGridPosList()                              // 初始化空间网格
findEnemysInNear(int xRange, int yRange, bool checkLookDirection)
findAllEnemysInNear(int xRange, int yRange, bool checkLookDirection)
RecalcAStarAttributes()                            // 物体移动后更新寻路
```

### 工作原理

```
物体移动 → 更新所在网格单元
             ↓
碰撞查询 → 只检查 nearGridPosList 中的邻近格子
             ↓
找到邻近对象 → 精确 AABB + Z 碰撞检测
```

避免全场景遍历，将碰撞查询复杂度从 O(n²) 降低到 O(k)（k = 邻近对象数）。

## 10. 标签系统

地图支持多种标签标记网格：

- **ExitTag** — 标记出口/传送点，触发房间切换
- **SpawnTag** — 怪物/NPC 生成点
- **GridTag** — 特殊网格属性（陷阱、事件触发等）

标签序列化在地图脚本中，`useExitTagForGridWhenSave` 控制保存行为。

## 整体架构图

```
┌─────────────────────────────────────────────────┐
│                 STDungeonScript                   │
│            副本定义（类型、迷宫列表）               │
└──────────────────┬──────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────┐
│                  MazeScript                       │
│         迷宫拓扑（房间网格、起点、Boss位置）        │
│    ┌──────────┬──────────┬──────────┐            │
│    │GridScript│GridScript│GridScript│  ← 房间    │
│    │  入口    │  普通    │  Boss    │            │
│    └──────────┴──────────┴──────────┘            │
└──────────────────┬──────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────┐
│              MapManager.LoadMap()                  │
│  ┌───────────────────────────────────────────┐   │
│  │ 渲染层: MapTile[] (基础+扩展瓦片)          │   │
│  ├───────────────────────────────────────────┤   │
│  │ 碰撞层: MapTileAttribute_Array (Byte[])   │   │
│  │         PassageRect 粗网格                 │   │
│  ├───────────────────────────────────────────┤   │
│  │ 寻路层: StaticGrid → Node[][]             │   │
│  │         A* + 启发式 + 对角移动             │   │
│  ├───────────────────────────────────────────┤   │
│  │ 实体层: Monster / NPC / PassiveObject     │   │
│  │         按网格坐标放置                     │   │
│  └───────────────────────────────────────────┘   │
└──────────────────┬──────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────┐
│            运行时空间分区                          │
│  nearGridPosList → 邻近对象快速查询                │
│  RecalcAStarAttributes() → 动态更新寻路           │
│  CheckEdgeCollision() → 边界约束                  │
└─────────────────────────────────────────────────┘
```

### 核心设计要点

- **多层网格分离** — 渲染、碰撞、寻路、空间分区各用独立网格，分辨率和更新频率不同
- **通行矩形** — 比瓦片更粗的碰撞网格，降低大地图内存开销
- **对象池化 GridPos** — 避免高频寻路产生的 GC 压力
- **多层瓦片地图** — `arrTilemapType` 支持 A/B 层切换（如地下通道）
- **迷宫拓扑** — MazeScript 定义房间连接，支持任务特定布局和隐藏房间
- **动态寻路更新** — 物体移动时 `RecalcAStarAttributes()` 实时更新可行走性
