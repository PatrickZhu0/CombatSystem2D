# 碰撞系统实现

## 核心数据结构：DNFCube（line 408071）

碰撞的基本单元是 **AABB 包围盒**，用 `DNFCube` 结构体表示：

```csharp
public struct DNFCube {
    public Vector3 start;       // 最小角
    public Vector3 end;         // 最大角
    public Vector3 realStart;   // 实际起始位置
    public Vector3 realEnd;     // 实际结束位置
    public Vector3 sizeVector;  // 宽/高/深 尺寸
}
```

碰撞判定方法：

```csharp
// 简单 AABB 重叠测试
isCollision(DNFCube other)

// 高级碰撞：支持偏移、翻转、攻击框扩展
IsCollision(Vector3 srcOffset, Boolean srcReverseVertical,
            DNFCube dest, Vector3 destOffset, Boolean destReverseVertical,
            ref Vector3 hitPos, Boolean applyAttackBoxExpandTest)
```

## 2.5D 深度（Z轴）碰撞

这是该系统最关键的设计。游戏是 2.5D，Z 轴模拟纵深（类似 DNF 的上下走位）。

### IRDCollisionObject（line 415358）

每个对象有 **两套包围体**：

```csharp
// 身体碰撞（受击）
protected Rect boundBox;         // XY 平面包围矩形
protected Vector2 zBound;        // (minZ, maxZ) 深度范围

// 攻击碰撞（打击）
protected Rect attackBoundBox;   // XY 平面攻击范围
protected Vector2 attackZBound;  // (minZ, maxZ) 攻击深度范围
protected DNFCube attackBoxCube; // 3D 攻击碰撞立方体
```

### 碰撞成立条件

XY 平面 AABB 重叠 **且** Z 范围重叠：

```
攻击者 attackZBound: [minZ_a, maxZ_a]
受击者 zBound:       [minZ_b, maxZ_b]

命中 = XY重叠 && maxZ_a >= minZ_b && minZ_a <= maxZ_b
```

这就是为什么 DNF 中在不同"行"的敌人打不到——Z 范围不重叠。

## 两阶段碰撞检测

### 粗筛（Broadphase）：网格空间分区

```csharp
// IRDCollisionObject 中
static List<GridPos> nearGridPosList;   // 预计算的邻近网格列表
InitNearGridPosList();                   // 初始化周围网格单元
```

通过网格快速排除远距离物体，只对邻近网格内的对象进行精确检测。

### 精检（Narrowphase）：AABB + Z 范围测试

核心方法 `checkHitCollisionBoxes()`（line 415779）：

```csharp
public static Boolean checkHitCollisionBoxes(
    IRDCollisionObject attacker,
    IRDCollisionObject damager,
    List<DNFCube> attackboxes,    // 攻击者的所有攻击框（当前帧）
    List<DNFCube> damageboxes,    // 受击者的所有受击框
    ref Vector3 hitPos)           // 输出命中位置（用于特效）
```

遍历所有攻击框 x 受击框组合，逐一做 AABB+Z 重叠测试，首次命中即返回。

## 攻击框 vs 受击框

### 攻击框（Hitbox）— 攻击者侧

- 存储在 `attackBoxCube`
- **逐帧变化** —— 由动画数据驱动，只在特定动画帧激活
- 有独立的 Z 深度范围 `attackZBound`

```csharp
// 动画帧驱动的攻击框
private DNFAnimationInfo cachendAnimationInfo;
private List<DNFCube> cachedAttackBoxes;      // 当前帧的攻击框列表
private DNFAnimationFrame cachedAnimationFrame;
private DNFCube cachedMaxAttackBoxCube;        // 最大包围框（快速剔除用）
```

### 受击框（Hurtbox）— 防御者侧

- 存储在 `boundBox`（Rect）+ `zBound`（Vector2）
- 相对固定，随角色移动

## 碰撞回调链

### 攻击侧完整流程

```
对象移动 updateMoveObject()
    ↓
位置合法性 checkMovablePos()
    ↓
碰撞检测 → 发现重叠对象
    ↓
onMoveCollision(axisIndex, collisionedObject)
    ↓
checkHitableTarget(damager)          ← 目标验证
  ├─ 是否免疫 (IsInvincible)
  ├─ 是否超级霸体 (IsSuperArmor)
  ├─ 是否同阵营 (isEnemy)
  ├─ 是否已被命中 (hitObjectList)
  └─ 穿透力检查 (checkPierce)
    ↓ 验证通过
OnBeforeAttack(damager, isStuck)     ← 攻击前回调
    ↓
onAttack(damager, attacker, isStuck) ← 执行命中
  ├─ 加入 hitObjectList（防重复命中）
  └─ hitObjectHitCountDic[target]++ （记录命中次数）
    ↓
OnAfterAttack(damager, isStuck)      ← 攻击后回调
```

### 受击侧流程

```
OnBeforeDamage(attacker, isStuck)    ← 受击前（免疫判断）
    ↓
onDamage(attacker, isCritical, isStuck) ← 执行受伤
    ↓
OnAfterDamage(attacker, isStuck)     ← 受击后（记录日志）
```

## 重复命中防护

```csharp
// IRDCollisionObject 中
public readonly List<IRDCollisionObject> hitObjectList;                    // 已命中目标列表
public readonly Dictionary<IRDCollisionObject, Int32> hitObjectHitCountDic; // 每目标命中次数
private Single hitObjectListClearTime;                                     // 清空时间点
```

同一次攻击不会重复命中同一目标。`hitObjectListClearTime` 到期后清空列表，允许多段技能的后续段命中。

## 子弹碰撞的特殊处理

### CNGunnerBullet（line 446879）onMoveCollision 流程

```

    ↓

    ↓

    ↓ 命中

    ↓

    ↓

    ↓

```

## 整体架构总结

```
┌─────────────────────────────────────────────────┐
│              每帧更新循环                         │
│  updateMoveObject() → 移动所有活动对象            │
└──────────────┬──────────────────────────────────┘
               ↓
┌──────────────────────────────────────┐
│  粗筛 (Broadphase)                    │
│  Grid 空间分区 → nearGridPosList      │
│  排除不在邻近网格的对象                 │
└──────────────┬───────────────────────┘
               ↓
┌──────────────────────────────────────┐
│  精检 (Narrowphase)                   │
│  checkHitCollisionBoxes()             │
│  ┌──────────────────────────────┐    │
│  │  XY 平面: AABB Rect 重叠     │    │
│  │  Z  轴:  zBound 范围重叠     │    │
│  │  动画帧: 当前帧攻击框有效？   │    │
│  └──────────────────────────────┘    │
└──────────────┬───────────────────────┘
               ↓
┌──────────────────────────────────────┐
│  目标验证 checkHitableTarget()        │
│  · 阵营检查 (isEnemy)                 │
│  · 免疫检查 (IsInvincible)            │
│  · 重复命中检查 (hitObjectList)        │
│  · 穿透力检查 (checkPierce)           │
└──────────────┬───────────────────────┘
               ↓
┌──────────────────────────────────────┐
│  伤害回调链                           │
│  OnBeforeAttack → onAttack            │
│    → OnAfterAttack                    │
│  OnBeforeDamage → onDamage            │
│    → OnAfterDamage                    │
└──────────────────────────────────────┘
```

### 核心设计要点

- **AABB + Z范围** 的 2.5D 碰撞方案，简单高效
- **动画帧驱动攻击框**，碰撞与动画严格同步
- **网格粗筛 + AABB精检** 两阶段降低开销
- **hitObjectList** 防重复命中 + `hitObjectListClearTime` 支持多段技能
- **分类位掩码** 快速过滤阵营/类型
