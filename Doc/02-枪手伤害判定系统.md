# 枪手伤害判定系统

## 整体伤害流水线

```
技能激活 → AttackInfo 构建 → 子弹/判定体生成 → 碰撞检测
    → 免疫检查 → 伤害计算 → 击退/异常状态 → 穿透判定
```

---

## 1. 伤害载荷：AttackInfo（line 361077）

所有伤害的核心数据结构：

```csharp
public Int32 maxHitCount           // 最大命中目标数
public Single power_               // 基础伤害倍率
public Single powerIncreaseRate    // 伤害成长系数
public Int32 absoluteDamage_       // 固定伤害值
public Single criticalHitRate_     // 暴击率
public Single stuckRate_           // 硬直/僵直率
public Int32 knockBackType_        // 击退类型 (0=无, 1=水平, 2=垂直)
public Int32 knockBackMovePixel_   // 击退距离
public Single weakness_rate_       // 属性弱点倍率
public Single finalDamageRate_     // 最终伤害修正（所有计算之后）
public Boolean xDefenseIgnore      // 无视防御标记
public List<ActiveStatusDatas> active_statuses_  // 附加异常状态
```

## 2. 子弹创建阶段

### 普通攻击流程

```
玩家按键 → onUserInputAttack()
  → procStateAttack()  // 进入攻击状态
    → CreateShootEffectAnimation()  // 在枪口帧创建子弹
```

子弹实例化时设置：

- `AttackInfo` = 枪械基础攻击 + 技能修正
- `pierceForce_` = `gunInfo.pierceForceOfGunnerGun_`（穿透力）
- 位置 = 枪口位置 + 方向偏移（`ATTACK_UP/DOWN/JUMP_MUZZLE_DISTANCE`）

## 3. 碰撞检测阶段

### CNGunnerBullet（line 446815）核心方法链

```
子弹移动 updateMoveValues()
    ↓
碰撞检测 onMoveCollision()     ← 空间重叠检测
    ↓
目标验证 checkHitableTarget()  ← 是否可被攻击
    ↓
穿透验证 checkPierce()         ← pierceForce_ vs 目标质量
    ↓
命中确认
```

### 副武器子弹 CNGunnerSubWeaponBullet（line 442047）

| 副武器 | 判定方式 | 特殊处理 |
|--------|---------|---------|
| 火焰喷射 | 持续判定 `proc_FlameThrower()` | DoT 伤害，有状态机 |
| Steyr 重步枪 | 命中/到期爆炸 `createSteyrExplosion()` | 范围伤害，`steyrExpSizeRate_` |
| FM31 导弹 | 范围爆炸 `createFM31Explosion()` | AoE，`fM31StuckRate_` 独立硬直率 |
| 散弹枪 | 近距扇形 `BarrelShotStart()` | `barrelShotMeleeRange` 判定范围 |

## 4. 伤害计算阶段

命中确认后，经过 **三层回调**：

```
[第一层] OnBeforeAttack()  — 子弹侧预处理
[第二层] Target.OnBeforeDamage()  — 目标侧免疫检查
         ↓ 返回 ENUM_IMMUNE_TYPE (NONE/PHYSICAL/MAGICAL/ALL)
         ↓ 若免疫则中止
[第三层] onAttack()  — 执行实际命中逻辑
```

### 伤害公式

```
基础伤害 = power_ × 物理攻击力 × powerIncreaseRate
绝对伤害 = absoluteDamage_
暴击倍率 = isCritical ? (1.0 + criticalDamageRate) : 1.0
属性伤害 = weakness_rate_ × weakness_power_

最终伤害 = (基础伤害 + 绝对伤害) × 暴击倍率 × finalDamageRate_

防御减免 = Target.Defense × DefenseRate + AbsoluteDefense
实际伤害 = 最终伤害 - 防御减免   (xDefenseIgnore=true 时跳过)
```

## 5. 穿透系统

枪手独有的核心机制：

```
子弹创建时: pierceForce_ = gunInfo.pierceForceOfGunnerGun_
                ↓
每次命中目标: procImprovePiercing()
                ↓
          pierceForce_ -= 目标抗性
                ↓
        pierceForce_ > 0 ? → 继续飞行，可命中下一个目标
        pierceForce_ ≤ 0 ? → 子弹销毁
```

穿透力决定了一颗子弹能打穿多少个敌人，是枪手 AoE 能力的关键参数。

## 7. 击退与异常状态

命中后效果应用：

```
onAttack() 执行后:
  ├── 创建命中粒子特效
  ├── 播放命中音效
  ├── 应用击退:
  │     knockBackType_ = 1 → 水平推开 (knockBackMovePixel_)
  │     knockBackType_ = 2 → 垂直击飞 (nUpForce)
  ├── 应用硬直: stuckRate_ × 硬直时长
  ├── 应用异常状态: active_statuses_[] 逐一应用
  │     (燃烧、中毒、冰冻等)
  └── 副武器特殊效果:
        SubWeaponStuckDownPowerRate   // 副武器击倒伤害率
        SubWeaponStuckDownStuckRate   // 副武器击倒硬直率
```

## 8. 技能特化伤害

### HeadShot（头射）

- 继承 `ShootSkill`，精准单体高倍率
- `CreateHeadShotEffect()` 创建专属命中特效
- 支持后撤头射 `CanBackstepHeadshot()`

### MultiHeadShot（多重头射）

- `GetMultiHeadShotTarget()` 扇形/环形选定多目标
- 每个目标独立 AttackInfo 计算
- 通过 `MULTI_HEAD_SHOT_OFFSETS[]`、`ROTATE_ANGLES[]` 控制弹道分布

### Gatling（加特林）

- 连射状态机：`gatlingShootNumber_` / `gatlingShootMaxNumber_`
- `gatlingShootTerm_` 控制每发间隔
- 每轮重新计算伤害

### GunnerDashAttack（冲刺攻击）

- 非投射物，碰撞体判定
- `changeBackForce_[]` 动态击退力度
- `DASH_ATTACK_STUN_DELAY` 僵直延迟

## 10. 完整示例：一次普攻的伤害判定

```
1.  按下攻击键 → onUserInputAttack()
2.  播放攻击动画 → procStateAttack()
3.  枪口帧触发 → CreateShootEffectAnimation() 实例化 CNGunnerBullet
4.  子弹飞行 → updateMoveValues() 每帧更新位置
5.  碰到敌人 → onMoveCollision() 检测碰撞
6.  验证目标 → checkHitableTarget() + checkPierce()
7.  预处理 → OnBeforeAttack() + Target.OnBeforeDamage() 免疫检查
8.  命中 → onAttack() 创建特效、音效
9.  算伤害 → power_ × 攻击力 × 暴击 × finalDamageRate_ - 防御
10. 应用效果 → 击退 + 硬直 + 异常状态
11. 穿透判定 → procImprovePiercing() 减少 pierceForce_
12. 后处理 → OnAfterAttack() + OnAfterDamage() 记录伤害日志
```

核心设计思路：通过 **AttackInfo 数据驱动** + **多层回调链** + **穿透力系统** 实现灵活的伤害判定，每个技能只需配置不同的 AttackInfo 参数即可获得差异化的伤害表现。
